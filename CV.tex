% ========== Tiny-RAG ==========
\textbf{Tiny-RAG 企业知识库系统} \hfill  \titlebreak
项目描述：设计并实现高可用、高召回的企业级混合检索知识库系统，解决传统RAG在专有名词识别、长文档语义割裂、大文件上传可靠性等场景的痛点，支持多租户安全访问与低Token消耗的智能问答。
\begin{itemize}[nosep, topsep=\intraProjectListTopSep]
    \item \textbf{混合检索引擎架构设计}：基于BM25（稀疏）与句向量Embedding（稠密）构建并行双引擎检索架构，结合KNN向量召回与关键词检索，提升专有名词与长尾语义的覆盖率；引入Cross-Encoder做高精度重排。
    \item \textbf{双引擎索引落地实现}：基于Elasticsearch + IK分词搭建BM25关键词索引，使用通义千问2048维高维Embedding做语义向量化并存入向量库，通过RRF算法并行融合检索结果并重排。
    \item \textbf{长文档语义优化方案}：提出Parent-Child Indexing（父子索引）策略，以小粒度子块定位检索、回溯父级段落恢复上下文，兼顾召回精度与语义完整性。
    \item \textbf{高可靠大文件上传架构}：基于Kafka解耦「上传→解析→向量化」全流程，支持分片上传与断点续传；利用Redis Bitmap跟踪分片状态，结合MinIO与分片MD5校验实现可靠合并。
    \item \textbf{智能检索优化与安全管控}：采用Dynamic Top-K调度算法，根据召回分数分布动态调整提交给LLM的片段数量；开发基于LLM的Query-Rewrite模块；引入RBAC多级权限控制。
\end{itemize}

\textbf{PaiAgent 企业级AI工作流可视化编排平台} \hfill  \titlebreak
项目描述：基于LangGraph4j与Spring AI构建企业级AI工作流平台，支持可视化拖拽编排多模型与工具节点，通过DAG引擎执行复杂AI任务，实现零代码快速构建AI Agent应用。
\begin{itemize}[nosep, topsep=\intraProjectListTopSep]

\end{itemize}
