\textbf{Tiny-RAG 企业知识库系统} \hfill 2025.10 -- 2026.02 \titlebreak
项目描述：设计并实现高可用、高召回的企业级混合检索知识库系统，解决传统RAG在专有名词识别、长文档语义割裂、大文件上传可靠性等场景的痛点，支持多租户安全访问与低Token消耗的智能问答，可直接落地企业内部知识问答、智能客服等场景。
\begin{itemize}[nosep, topsep=\intraProjectListTopSep]
    \item \textbf{混合检索引擎架构设计}：基于BM25（稀疏）与句向量Embedding（稠密）构建并行双引擎检索架构，结合KNN向量召回与关键词检索，提升专有名词与长尾语义的覆盖率；引入Cross-Encoder做高精度重排，大幅提升上下文相关性与排序准确性。
    \item \textbf{双引擎索引落地实现}：基于Elasticsearch + IK分词搭建BM25关键词索引，使用通义千问2048维高维Embedding做语义向量化并存入向量库，通过RRF算法并行融合检索结果并重排，输出高相关候选片段。
    \item \textbf{长文档语义优化方案}：提出Parent-Child Indexing（父子索引）策略，以小粒度子块定位检索、回溯父级段落恢复上下文，兼顾召回精度与语义完整性，解决长文档切片导致的语义割裂问题。
    \item \textbf{高可靠大文件上传架构}：基于Kafka解耦「上传→解析→向量化」全流程，支持分片上传与断点续传；利用Redis Bitmap跟踪分片状态，结合MinIO与分片MD5校验实现可靠合并，提升上传可靠性与系统吞吐能力。
    \item \textbf{智能检索优化与安全管控}：采用Dynamic Top-K调度算法，根据召回分数分布动态调整提交给LLM的片段数量，显著降低Token消耗；开发基于LLM的Query-Rewrite模块，完成指代消解与语义扩展；引入RBAC多级权限控制，支持租户隔离与文档级鉴权。
\end{itemize}
